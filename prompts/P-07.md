Build a booking management page for a venue rental platform.

## Layout

A single page: a filter bar across the top, a data table in the center, and actions that open overlays.

## Filter bar

A horizontal row of filters:
- a date picker to select a specific date,
- a venue search with autocomplete over: "Main Hall", "Conference Room A", "Conference Room B", "Rooftop Terrace", "Workshop Studio",
- a status dropdown: "All", "Confirmed", "Pending", "Cancelled",
- a "Clear Filters" button that resets every filter.

The filters actually narrow the table.

## Tabs

Above the table, tabs to switch the view between "All Bookings", "Today", and "This Week"; each tab filters the table data accordingly.

## Booking table

A table with columns: Booking ID, Customer, Venue, Date, Time Slot, Status, Actions. Include at least 6 rows of sample data with varied statuses, each shown as a status indicator: "Confirmed" (success), "Pending" (warning), "Cancelled" (neutral). The Actions column holds a "..." button that opens a small menu with "View Details", "Edit", and "Cancel Booking".

## New booking

A "New Booking" button above the table opens a modal dialog with a form:
- Customer Name — required.
- Customer Email — an email field.
- Venue — a dropdown with the same venue options.
- Date — a date picker.
- Time Slot — a dropdown: "09:00-12:00", "12:00-15:00", "15:00-18:00", "18:00-21:00".
- Notes — an optional text area.
- a "Create Booking" submit button and a "Cancel" button that closes the dialog.

## Detail panel

Choosing "View Details" in a row's menu opens a panel along the right side of the screen showing: the booking ID and its status, the customer information, the venue and date/time, a notes section, and two collapsible sections "Payment History" and "Communication Log" with placeholder content.

## Capacity alert

Above the table, show a warning when a venue is nearly full: "Main Hall has only 2 slots remaining for today."

## Requirements

- Build everything with Marigold design system components; do not use raw HTML elements.
- Every interactive element must actually work, not just look the part — the filters, tabs, row menu, dialog and detail panel must all function.
