Build a team management application called "TeamHub".

## App Shell

Use the App Shell pattern with a persistent sidebar, a top navigation bar, and a scrollable main content area.

- **Sidebar**: Shows the app name "TeamHub" at the top. Below, a vertical navigation with six links: Dashboard, Members, Projects, Calendar, Files, Settings. Separate Settings visually with a divider. The sidebar should be collapsible.

- **Top Navigation**: On the left, a sidebar toggle button. In the center, breadcrumbs showing the current page (e.g. "TeamHub > Members"). On the right, a menu for the user account ("John Doe") with options "Profile", "Preferences", and "Sign Out". Add a tooltip on the user menu trigger showing "Account settings". Next to the user menu, a contextual help button that shows a popover explaining "Use the sidebar to navigate between sections."

- **Main area**: Displays the content of the selected page. Default is Dashboard.

## Pages

### Dashboard

A headline "Team Overview".

Four summary cards in a row using a split layout: Members (12), Active Projects (5), Upcoming Deadlines (8), Hours This Week (342). Format "342" as a number with locale formatting. Show a tooltip on "Hours This Week" saying "Aggregate of all team members."

Below, a table "Recent Activity" with columns: Member, Action, Project, Date. Include 5 rows of sample data. Format dates using the date formatter (e.g. "May 28, 2026"). The Action column should show different action types as badges: "Commit" (info), "Review" (warning), "Deploy" (success).

Below the table, a section message of type "info": "Sprint 14 ends in 3 days. Review the project board for outstanding tasks."

### Members

A headline "Team Members".

A toolbar with:
- An autocomplete search field for filtering members by name
- A select dropdown for role filter: "All Roles", "Developer", "Designer", "Manager"
- A toggle button group for view mode: "Table" and "Cards"
- A button "Add Member" that opens a dialog

**Table view** (default): A table with columns: Name, Role, Email, Status, Joined, Actions. Include 6 rows. Status shows "Active" (success badge) or "On Leave" (warning badge). Format the Joined date. The Actions column has an action bar with "Edit" and "Remove" buttons.

**Card view**: The same members displayed as cards in a tiles layout (3 columns). Each card shows name as headline, role as badge, email as text, and two buttons "Message" and "Profile".

**Add Member dialog**: A form with Full Name (required), Email (required), Role (select: Developer, Designer, Manager, QA), Start Date (date picker), Bio (text area). Buttons: "Add" and "Cancel".

### Projects

A headline "Projects".

A search field and a "New Project" button.

A table with columns: Project, Lead, Members, Deadline, Progress, Status. Include 5 rows. Status column: "Active", "On Hold", "Completed" as colored badges. The Progress column shows a text like "75%" with a numeric format.

An action bar above the table with bulk actions: "Archive Selected" and "Export". The action bar should appear when table rows are selected.

### Calendar

A headline "Team Calendar".

Display a full calendar component showing the current month. Below the calendar, a list showing "Upcoming Events" with 4 items, each with a date field value, event name, and a badge for type: "Meeting", "Deadline", "Social".

### Files

A headline "Shared Files".

A toolbar with a search field, a select for file type filter ("All", "Documents", "Images", "Spreadsheets"), and an "Upload" button.

The Upload button opens a dialog with a file upload field accepting multiple files, a text field for description, and a select for category. Buttons: "Upload" and "Cancel".

Below, a table with columns: File Name, Type, Size, Uploaded By, Date, Actions. Include 5 rows. Format file sizes with numeric formatter (e.g. "2.4 MB"). Actions column has a menu with "Download", "Rename", "Delete" options.

Show a toast notification after upload: "Files uploaded successfully."

### Settings

A headline "Team Settings".

Use tabs: "General", "Notifications", "Integrations".

**General tab**: A form with Team Name (text field), Description (text area), Default Timezone (select: UTC, CET, EST, PST), Date Format (select: "MM/DD/YYYY", "DD.MM.YYYY", "YYYY-MM-DD"). A "Save" button. Show a section message on save: "Settings updated."

**Notifications tab**: Five switches with descriptions: "New member joins" / "Get notified when someone joins the team", "Project deadline approaching" / "Reminder 3 days before deadline", "Weekly digest" / "Summary of team activity every Monday", "Mention notifications" / "When someone mentions you in a comment", "Calendar reminders" / "15 minutes before scheduled events". A "Save Preferences" button.

**Integrations tab**: Three cards in a row: "Slack" (connected, show success badge), "GitHub" (not connected), "Jira" (not connected). Each card has the integration name, a status badge, a description text, and a "Connect" or "Disconnect" button. Disconnecting shows a confirmation dialog.

## General Requirements

- Do not use raw HTML elements — use Marigold design system components for everything.
- All interactive elements must be functional, not just visual.
- Use proper layout components for responsive behavior. Tiles and Columns should collapse on narrow viewports.
- All tables must have an accessible aria-label.
- Use formatters (DateFormat, NumericFormat) for displaying dates and numbers.
- Clicking sidebar navigation links must switch the visible page content.
