Build a dashboard application for managing events.

The app should have a sidebar navigation, a main content area that switches between different pages, and feel like a real admin tool.

The sidebar should show the app name, navigation links for different sections, and a way for the user to access their account or sign out. Consider showing the user where they are in the app at all times.

Include the following pages:

- **Overview**: A welcome area, summary statistics displayed prominently, and a data table showing upcoming events with their statuses.
- **Events**: A list of events that can be searched and filtered. Users should be able to create new events through an overlay form that collects the event name, date, location, capacity, and description.
- **Attendees**: A searchable attendee list showing name, email, events attended, and an activity status.
- **Reports**: Multiple report views that the user can switch between — revenue stats, attendance stats with a top-events table, and a summary view with expandable quarterly breakdowns.
- **Settings**: Organized into sections — general settings with form fields and dropdowns, and a notification section with on/off toggles for different notification types.

Provide additional context or explanations where numbers or stats might be unclear. Show contextual feedback messages where appropriate (welcome messages, success confirmations, warnings).

Use Marigold design system components for everything — no raw HTML elements.
