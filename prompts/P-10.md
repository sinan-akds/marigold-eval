Build a team management application called "TeamHub".

## Shell

An application shell with a persistent sidebar, a top navigation bar, and a scrollable main content area.

- **Sidebar** — the app name "TeamHub" at the top, then a vertical navigation with six entries: Dashboard, Members, Projects, Calendar, Files, Settings. Separate "Settings" with a dividing line. The sidebar can be collapsed and expanded.
- **Top navigation** — on the left, a button to toggle the sidebar; in the center, a breadcrumb for the current page (e.g. "TeamHub > Members"); on the right, a menu for the user account ("John Doe") with "Profile", "Preferences", "Sign Out", a tooltip "Account settings" on its trigger, and beside it a help button that opens a popover "Use the sidebar to navigate between sections."
- **Main area** — shows the selected page (default Dashboard); the active navigation entry is clearly marked and choosing an entry switches the visible page.

## Dashboard

A heading "Team Overview". Four summary cards evenly spaced in a row: Members, Active Projects (5), Upcoming Deadlines (8), Hours This Week (342, shown with locale number formatting); the Members card shows the current number of team members and the "Hours This Week" card has a tooltip "Aggregate of all team members." Below, a table "Recent Activity" (Member, Action, Project, Date; 5 rows) with dates shown in a readable localized form (e.g. "May 28, 2026") and the Action column showing types as indicators: "Commit" (info), "Review" (warning), "Deploy" (success). Below the table, an info banner: "Sprint 14 ends in 3 days. Review the project board for outstanding tasks."

## Members

A heading "Team Members" and a toolbar with: a search field that filters members by name, a role filter ("All Roles", "Developer", "Designer", "Manager"), a control to switch the view between a table and cards, and an "Add Member" button that opens a dialog.

Start with 6 members. The view, search and role filter all work together, and the chosen view mode persists while navigating away and back.

- **Table view** (default): columns Name, Role, Email, Status, Joined, Actions; Status is "Active" (success) or "On Leave" (warning); Joined dates shown localized. Each row's Actions offer "Edit" and "Remove".
- **Card view**: the same members as cards, three per row (collapsing on narrow screens), each showing the name, the role as an indicator, the email, and "Message" and "Profile" buttons.

Behavior that must actually work end to end:
- **Add Member** opens a dialog (Full Name required, Email required, Role one of Developer / Designer / Manager / QA, Start Date, Bio) with "Add" and "Cancel". Adding appends the member to both views and increases the Members count on the Dashboard.
- **Edit** opens the same form pre-filled and updates the member in place.
- **Remove** asks for confirmation, then deletes the member from both views and updates the Dashboard count.
- Clicking a member (row or card) opens a detail panel along the side showing their full information.

## Projects

A heading "Projects", a search field, and a "New Project" button. A table with columns Project, Lead, Members, Deadline, Progress, Status (5 rows); Status is "Active", "On Hold" or "Completed" as indicators; Progress shows a percentage (e.g. "75%"). Rows are selectable; selecting one or more reveals a bar of bulk actions with "Archive Selected" and "Export". "Archive Selected" removes the selected projects from the table.

## Calendar

A heading "Team Calendar". A full calendar for the current month, and below it "Upcoming Events" with 4 items, each a date, an event name, and an indicator for its type: "Meeting", "Deadline", "Social".

## Files

A heading "Shared Files" and a toolbar with a search field, a file-type filter ("All", "Documents", "Images", "Spreadsheets"), and an "Upload" button. "Upload" opens a dialog with a file field that accepts multiple files, a description text field, and a category selector, plus "Upload" and "Cancel". A table with columns File Name, Type, Size, Uploaded By, Date, Actions (5 rows); sizes shown with number formatting (e.g. "2.4 MB"); each row's Actions menu offers "Download", "Rename", "Delete". After an upload, show a toast "Files uploaded successfully."

## Settings

A heading "Team Settings" and tabs "General", "Notifications", "Integrations".
- **General** — Team Name, Description, Default Timezone (UTC, CET, EST, PST), Date Format ("MM/DD/YYYY", "DD.MM.YYYY", "YYYY-MM-DD") and a "Save" button; saving shows "Settings updated." Changing the Team Name updates the name shown in the application.
- **Notifications** — five toggles, each with a description: "New member joins" / "Get notified when someone joins the team"; "Project deadline approaching" / "Reminder 3 days before deadline"; "Weekly digest" / "Summary of team activity every Monday"; "Mention notifications" / "When someone mentions you in a comment"; "Calendar reminders" / "15 minutes before scheduled events". A "Save Preferences" button.
- **Integrations** — three cards in a row: "Slack" (connected, success indicator), "GitHub" (not connected), "Jira" (not connected). Each card shows the name, the status, a description, and a "Connect" or "Disconnect" button. Disconnecting asks for confirmation first.

## Requirements

- Build everything with Marigold design system components; do not use raw HTML elements.
- Every interactive element must actually work, not just look the part. In particular, member add / edit / remove must stay consistent across the table view, the card view and the Dashboard count; project bulk actions must respond to the row selection; and every search and filter must narrow its data live.
- Dates and numbers are shown using proper localized formatting.
- Card grids collapse to a single column on narrow screens.
- Every table must carry an accessible label.
