Build a multi-step event registration form with four steps.

## Frame

A card holds the whole flow. At the top, the current step number and title (e.g. "Step 2 of 4 — Event Details"). Below it, the fields for the current step. At the bottom, navigation: "Back" (disabled on step 1) and "Next" — which becomes "Submit Registration" on step 4.

## Step 1 — Personal information

- Full Name — required.
- Email — required, validated as an email address.
- Phone Number — optional.
- Company / Organization — optional.
- Job Title — optional, with autocomplete suggestions: "Developer", "Designer", "Product Manager", "Engineering Manager", "CTO", "Other".
- Profile Photo — a file upload that accepts images only.
- An info banner: "Your information will only be used for this event."

## Step 2 — Event details

- Event Date — a date picker, required.
- Preferred Time Slot — a time field.
- Session Track — a choice between "Technical", "Design", "Business", "Workshop".
- Dietary Requirements — a searchable selector over "None", "Vegetarian", "Vegan", "Gluten-Free", "Kosher", "Halal" that also allows a custom typed value.
- Number of Guests — numeric, from 0 to 5.
- Special Requests — an optional text area.

## Step 3 — Preferences

- T-Shirt Size — a dropdown: "XS", "S", "M", "L", "XL", "XXL".
- Topics of Interest — a field to add topics as removable tags, drawn from suggestions "AI/ML", "Web Development", "Cloud", "Security", "DevOps", "Mobile", "Data Science".
- Communication Preferences — a group of checkboxes: "Email updates about the event", "SMS reminders", "Post-event survey", "Newsletter subscription".
- Accessibility Needs — a toggle "I have accessibility requirements"; when on, reveal a text area for details.

## Step 4 — Review & confirm

Show everything entered, grouped in collapsible sections:
- "Personal Information" — name, email, phone, company, job title.
- "Event Details" — date, time, track, dietary, guests.
- "Preferences" — t-shirt size, topics, communication preferences.

Below them: a required checkbox "I agree to the terms and conditions" and a "Submit Registration" button (enabled only once the checkbox is ticked).

## After submission

Replace the form with a success view: briefly show a loading indicator (about one second), then a success banner "Registration confirmed!", a summary card with the registrant's name, email, event date and a generated confirmation number, a "Register Another" button that resets the flow, and a toast "Registration submitted successfully".

## Requirements

- Build everything with Marigold design system components; do not use raw HTML elements (an image tag for the uploaded photo preview is the only exception).
- Form state must persist across steps — going back preserves entered values.
- Required fields are validated before the user may move to the next step.
- Every interactive element must actually work, not just look the part.
