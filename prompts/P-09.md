Build a multi-step event registration form with four steps.

## General Layout

A card containing the registration flow. At the top, show the current step number and title (e.g. "Step 2 of 4 — Event Details"). Below the step indicator, show the form content for the current step. At the bottom, navigation buttons: "Back" (disabled on step 1) and "Next" (or "Submit Registration" on step 4).

## Step 1: Personal Information

- Full Name (required text field)
- Email (required text field)
- Phone Number (optional text field)
- Company / Organization (optional text field)
- Job Title (optional autocomplete field with suggestions: "Developer", "Designer", "Product Manager", "Engineering Manager", "CTO", "Other")
- Profile Photo (file upload field, accepts images only)
- A section message of type "info": "Your information will only be used for this event."

## Step 2: Event Details

- Event Date (date picker, required)
- Preferred Time Slot (time field)
- Session Track: a radio group with options "Technical", "Design", "Business", "Workshop"
- Dietary Requirements: a combo box where users can search and select from: "None", "Vegetarian", "Vegan", "Gluten-Free", "Kosher", "Halal" — or type a custom value
- Number of Guests (number field, min 0, max 5)
- Special Requests (text area, optional)

## Step 3: Preferences

- T-Shirt Size: a select dropdown with options "XS", "S", "M", "L", "XL", "XXL"
- Topics of Interest: a tag field where users can add tags from suggestions "AI/ML", "Web Development", "Cloud", "Security", "DevOps", "Mobile", "Data Science"
- Communication Preferences: a checkbox group with options:
  - "Email updates about the event"
  - "SMS reminders"
  - "Post-event survey"
  - "Newsletter subscription"
- Accessibility Needs: a switch labeled "I have accessibility requirements" — when enabled, show a text area for details

## Step 4: Review & Confirm

Display all entered information organized in an accordion with sections:
- "Personal Information" — showing name, email, phone, company, job title
- "Event Details" — showing date, time, track, dietary, guests
- "Preferences" — showing t-shirt size, topics, communication prefs

Below the accordion:
- A checkbox: "I agree to the terms and conditions" (required to submit)
- A "Submit Registration" button

## After Submission

Replace the form with a success view:
- Show a loader briefly (1 second simulated delay)
- Then show a section message of type "success": "Registration confirmed!"
- Display a summary card with the registrant's name, email, event date, and a confirmation number (generate a random ID)
- A "Register Another" button to reset the form
- Show a toast notification: "Registration submitted successfully"

## Requirements

- Do not use raw HTML elements. Use Marigold design system components for everything.
- Form state must persist across steps — going back should preserve entered values.
- Validate required fields before allowing navigation to the next step.
- All interactive elements must be functional, not just visual.
