Build a complete event management dashboard called "EventHub".

## Layout

The app should have a persistent sidebar on the left spanning the full height, a top bar across the top, and a scrollable main content area.

- **Sidebar**: Shows the app name "EventHub" at the top. Below that, a vertical navigation with five links: Dashboard, Events, Attendees, Reports, and Settings. Separate Settings visually from the other four items with a divider. Include a "Help & Support" link at the bottom of the sidebar. The sidebar should be collapsible with a toggle button.

- **Top bar**: On the left, a button to toggle the sidebar. In the center, breadcrumb navigation showing the current page (e.g. "EventHub > Dashboard"). On the right, a dropdown menu for the user account with options "Profile" and "Sign out". Clicking "Sign out" should show a confirmation dialog asking "Are you sure you want to sign out?" with "Cancel" and "Sign out" buttons.

- **Main area**: Displays the content of whichever page is currently selected in the sidebar.

## Pages

Clicking a sidebar link should switch the main content area to show the corresponding page. The default page is "Dashboard".

### Dashboard

A heading "Dashboard Overview".

Show an info section message at the top: "Welcome back! You have 3 events starting this week."

Below that, four summary cards in a single row: Total Events (24), Tickets Sold (1,849), Revenue ($45,230), Upcoming (8). Each card shows a label and a large value. Add a tooltip to the "Revenue" card that says "Net revenue after fees and refunds".

Below the cards, a table titled "Upcoming Events" with columns: Event, Date, Venue, Tickets Sold, Status. Include at least 6 rows with realistic example data. Show statuses like "On Sale", "Draft", and "Sold Out" as colored badges.

### Events

A heading "Events", a search input for filtering by name or location, and a "Create Event" button.

Below that, a table with columns: Name, Date, Location, Capacity, Status. Include at least 6 rows of sample data. The table should be filterable by the search input.

Clicking "Create Event" should open a dialog with a form containing: Event Name (text field, required), Date (date picker), Location (text field), Capacity (number field), Description (text area), and a "Create" submit button and a "Cancel" button. The dialog should close when either button is pressed.

### Attendees

A heading "Attendees", a search input for filtering by name or email, and the attendee count displayed as text (e.g. "6 attendees").

A table with columns: Name, Email, Events Attended, Last Active, Status. Include at least 6 rows of sample attendee data. The Status column should show "Active" (success badge) or "Inactive" (warning badge) based on whether the last active date is within the last 30 days. The table should be filterable by the search input.

### Reports

A heading "Reports".

Use tabs to switch between three report views: "Revenue", "Attendance", and "Overview".

- **Revenue tab**: Show four summary cards in a row — Total Revenue ($45,230), This Month ($8,420), Average per Event ($1,885), Refunds ($1,230). Below the cards, show a section message of type "success" saying "Revenue is up 12% compared to last month."

- **Attendance tab**: Show four summary cards — Total Attendees (3,200), Repeat Visitors (890), Average per Event (178), No-shows (145). Below the cards, show a table titled "Top Events by Attendance" with columns: Event, Date, Attendees, Capacity, Fill Rate. Include at least 4 rows.

- **Overview tab**: Show a text summary paragraph and an accordion with three expandable sections: "Q1 Summary", "Q2 Summary", and "Q3 Summary". Each section should contain a short paragraph of text summarizing that quarter's performance.

### Settings

A heading "Settings".

Use tabs to switch between two settings sections: "General" and "Notifications".

- **General tab**: A form with: Organization Name (text field), Contact Email (email text field), Default Currency (dropdown with options: USD, EUR, GBP), Default Timezone (dropdown with options: UTC, CET, EST, PST), and a "Save Changes" button. Show a success section message "Settings saved successfully." after the form is submitted.

- **Notifications tab**: A form with four switches: "Email Notifications", "SMS Notifications", "Weekly Digest", and "Marketing Emails". Include a description text below each switch explaining what it controls. Add a "Save Preferences" button.

## General requirements

- Use proper spacing and consistent layout throughout.
- Do not use raw HTML elements — use Marigold design system components for everything.
- All interactive elements (buttons, links, form fields) must be functional, not just visual.
- All tables should have an accessible `aria-label`.
- The sidebar active state should clearly indicate which page is currently selected.
