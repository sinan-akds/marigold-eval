Build a ticket shop page for a live event platform.

A customer visits this page to purchase tickets for a specific event. The page should support the full purchase flow:

1. Display the event information (name, date, venue, a short description)
2. Show available ticket categories with their prices — some categories may be sold out
3. Let the customer select how many tickets they want per category
4. Collect buyer information (name, email, and any other details you think are relevant)
5. Show an order summary with line items and a total before the customer confirms
6. After submission, show a confirmation or error state

The page should feel polished and production-ready. Handle edge cases gracefully — sold-out categories, empty selections, validation errors, and loading or processing states. Use appropriate visual hierarchy, spacing, and feedback mechanisms.

Do not use raw HTML elements. Use Marigold design system components for all UI elements.
