Build a product catalog page for an online merchandise store.

## Page Header

A headline "Merchandise Store" and a short text description "Browse our collection of branded merchandise."

## Toolbar

A horizontal bar with:
- A search field for filtering products by name
- A select dropdown for sorting: "Newest", "Price: Low to High", "Price: High to Low", "Most Popular"
- A button labeled "Filters" that opens a drawer from the right side

## Filter Drawer

The drawer contains filter controls:
- **Category**: A checkbox group with options "T-Shirts", "Hoodies", "Accessories", "Posters", "Stickers"
- **Price Range**: A slider from $0 to $100
- **Size**: A checkbox group with options "XS", "S", "M", "L", "XL"
- **Availability**: A switch labeled "In stock only"
- An "Apply Filters" button and a "Reset" button

## Applied Filters

Below the toolbar, show applied filters as removable tags. Include a "Clear all" action when filters are active. When no filters are applied, show italic text "No filters applied".

## Product Grid

Display 8 product cards in a tile layout (4 columns on desktop, 2 on tablet, 1 on mobile). Each card contains:
- Product name as a headline
- Price displayed prominently (e.g. "$29.99")
- A badge for status: "New" (info), "Sale" (warning), or "Sold Out" (none/neutral)
- A short description text (one sentence)
- An "Add to Cart" button (disabled for sold-out items)

Two of the 8 products should be marked as "Sold Out" with disabled buttons.

## Empty State

When filters match no products, show an empty state with:
- Title: "No products found"
- Description: "Try adjusting your filters or search query."
- An action button: "Clear all filters"

## Pagination

Below the product grid, a pagination component showing "Page 1 of 3" with previous/next navigation.

## Requirements

- Do not use raw HTML elements. Use Marigold design system components for everything.
- Use Tiles for the product grid with responsive column counts.
- Use the Drawer component for the filter panel.
- Use Tag.Group for applied filters with onRemove support.
- All interactive elements must be functional, not just visual.
