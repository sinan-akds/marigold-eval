Build a booking management page for a venue rental platform.

## Layout

A single page with a filter bar at the top, a data table in the center, and action capabilities via overlays.

## Filter Bar

A horizontal row of filters:
- Date filter: a DatePicker for selecting a specific date
- Venue search: a ComboBox with autocomplete for venue names (options: "Main Hall", "Conference Room A", "Conference Room B", "Rooftop Terrace", "Workshop Studio")
- Status filter: a Select dropdown with options "All", "Confirmed", "Pending", "Cancelled"
- A "Clear Filters" button to reset all filters

## Booking Table

A table with columns: Booking ID, Customer, Venue, Date, Time Slot, Status, Actions.

Include at least 6 rows of sample data with varied statuses:
- "Confirmed" — success badge
- "Pending" — warning badge
- "Cancelled" — none/neutral badge

The Actions column has a Menu (triggered by a "..." button) with options: "View Details", "Edit", "Cancel Booking".

## Create Booking Dialog

A "New Booking" button above the table. Clicking it opens a Dialog with a form:
- Customer Name (required text field)
- Customer Email (text field with type email)
- Venue (Select dropdown with the same venue options)
- Date (DatePicker)
- Time Slot (Select with options: "09:00-12:00", "12:00-15:00", "15:00-18:00", "18:00-21:00")
- Notes (TextArea, optional)
- "Create Booking" submit button and "Cancel" button

## Detail Drawer

Clicking "View Details" in the row menu opens a Drawer panel on the right showing:
- Booking ID and status badge
- Customer information
- Venue and date/time details
- Notes section
- An expandable Accordion with sections "Payment History" and "Communication Log" (placeholder content)

## Tabs for Views

Above the table, Tabs to switch between "All Bookings", "Today", and "This Week". Each tab filters the table data accordingly.

## Capacity Alert

Show a SectionMessage of type "warning" above the table when a venue is nearly fully booked: "Main Hall has only 2 slots remaining for today."

## Requirements

- Use Table with Table.Header, Table.Body, Table.Row, Table.Cell for the data table.
- Use DatePicker for date selection, ComboBox for venue search.
- Use Dialog with Dialog.Trigger for the create form.
- Use Drawer for the detail view.
- Use Tabs with Tabs.List and Tabs.TabPanel for view switching.
- Use Menu with Menu.Item for row actions.
- Use Badge for status indicators.
- Use Accordion with Accordion.Item for detail sections.
- Use SectionMessage for alerts.
- Use Headline and Text for typography.
- Use Stack and Columns for layout.
- Buttons must use onPress, not onClick.
- Do not use raw HTML elements. Use Marigold design system components for everything.
