Build a booking management page for a venue rental platform.

## Layout

A single page with a filter bar at the top, a data table in the center, and action capabilities via overlays.

## Filter Bar

A horizontal row of filters:
- Date filter: a date picker for selecting a specific date
- Venue search: a search field with autocomplete for venue names (options: "Main Hall", "Conference Room A", "Conference Room B", "Rooftop Terrace", "Workshop Studio")
- Status filter: a dropdown with options "All", "Confirmed", "Pending", "Cancelled"
- A "Clear Filters" button to reset all filters

## Booking Table

A data table with columns: Booking ID, Customer, Venue, Date, Time Slot, Status, Actions.

Include at least 6 rows of sample data with varied statuses:
- "Confirmed" — success badge
- "Pending" — warning badge
- "Cancelled" — none/neutral badge

The Actions column has a context menu (triggered by a "..." button) with options: "View Details", "Edit", "Cancel Booking".

## Create Booking Dialog

A "New Booking" button above the table. Clicking it opens a modal dialog with a form:
- Customer Name (required text field)
- Customer Email (text field with type email)
- Venue (dropdown with the same venue options)
- Date (date picker)
- Time Slot (dropdown with options: "09:00-12:00", "12:00-15:00", "15:00-18:00", "18:00-21:00")
- Notes (multiline text area, optional)
- "Create Booking" submit button and "Cancel" button

## Detail Panel

Clicking "View Details" in the row menu opens a side panel on the right showing:
- Booking ID and status badge
- Customer information
- Venue and date/time details
- Notes section
- Expandable sections for "Payment History" and "Communication Log" (placeholder content)

## View Tabs

Above the table, tabs to switch between "All Bookings", "Today", and "This Week". Each tab filters the table data accordingly.

## Capacity Alert

Show a warning message above the table when a venue is nearly fully booked: "Main Hall has only 2 slots remaining for today."

## Requirements

- Do not use raw HTML elements. Use Marigold design system components for everything.
- All interactive elements must be functional, not just visual.
