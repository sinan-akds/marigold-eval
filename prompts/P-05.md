Build a ticket shop page for a live event called "Summer Music Festival 2026".

## Event Information

Display the event details at the top:
- Event name: "Summer Music Festival 2026" as a headline
- Date: "July 15-17, 2026"
- Venue: "Stadtpark Freiburg"
- A short description: "Three days of live music featuring local and international artists."

## Ticket Categories

Show three ticket categories in cards:
1. **Early Bird** — $49, limited availability (show "Only 12 left" text)
2. **Regular** — $79, available
3. **VIP** — $149, sold out (visually indicate this, disable selection)

Each card shows the category name, price, a short description, and a number input for quantity selection. Sold-out categories should be clearly disabled with a "Sold Out" badge.

## Buyer Information Form

Below the ticket selection, a form collecting:
- Full Name (required text field)
- Email (required text field with type email)
- Phone Number (optional text field)

## Order Summary

Show selected tickets as line items with quantities and subtotals. Display the total price prominently. Include a "Purchase Tickets" submit button.

## Feedback

After submission, show a success message confirming the order. If no tickets are selected, show a warning preventing submission.

## Requirements

- Do not use raw HTML elements. Use Marigold design system components for everything.
- All interactive elements must be functional, not just visual.
