Build a settings page for an application.

The settings are grouped into four clearly separated, collapsible sections (an accordion).

## Profile (section 1)

- Display Name — required text field.
- Email Address — text field, validated as an email address.
- Bio — optional text area.
- A "Save Profile" button. Saving shows a brief confirmation.

## Appearance (section 2)

- Theme — a choice between "Light", "Dark", and "System Default".
- Language — a dropdown: "English", "German", "French".
- Compact Mode — a toggle, described as "Reduce spacing and font size for denser layouts".

## Notifications (section 3)

- A group of four checkboxes:
  - "Email notifications for new messages"
  - "Push notifications for mentions"
  - "Weekly activity digest"
  - "Marketing and promotional emails"
- Notification Sound — a dropdown: "Default", "Chime", "None".
- A "Save Notification Preferences" button.

## Privacy (section 4)

- Profile Visibility — a choice between "Public", "Team Only", "Private".
- Show Online Status — a toggle.
- Allow Search Engines — a toggle, described as "Let search engines index your public profile". This option only applies when the profile is Public: when visibility is "Team Only" or "Private", disable it.
- A "Save Privacy Settings" button.

## Requirements

- Build everything with Marigold design system components; do not use raw HTML elements.
- Every interactive element must actually work, not just look the part — including the dependency between profile visibility and the search-engine option.
